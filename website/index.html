Safe sandboxes for living code

Monitor your AI agents across worktrees

biomelab is a terminal dashboard for managing git worktrees, coding agents, and Docker Sandboxes across all your repositories.

  • Multi-repo visibility
  • Sandboxed isolation
  • Live PR and CI status
$ brew install mdelapenya/tap/biomelab

The Problem

Agent sprawl is real

You're running Claude on a feature branch, Copilot on a bug fix, and Codex on a refactor, all in the same repo. Which branch is which? Is anything dirty? Did that PR pass CI? You alt-tab between terminals trying to piece it together.
The Dashboard

One dashboard, every worktree

Register all your repositories. See every branch, every agent, every PR, updating in real time.

📂

Multi-repo

Register all your repositories. Switch between them instantly from the left panel tree.

🤖

Auto-detect agents

Claude, Copilot, Codex, Kiro, OpenCode, Gemini. Detected by process scanning, shown on each card.

💻

Auto-detect IDEs

VS Code, Cursor, Zed, Windsurf, GoLand, IntelliJ, PyCharm, Neovim, Vim. Tracked per worktree.

⚡

Live status

Dirty/clean, sync status, PR/MR with CI checks. Refreshed automatically every 5 seconds.

Docker Sandboxes

Isolation by default

Each worktree gets its own isolated Docker environment: filesystem, Docker daemon, and network. Agents can install packages, build containers, and modify files without touching your host.

🔒

Isolated Environments

One sandbox VM per agent per repo. All worktrees share it, isolated from your host and from other agents.

🧩

One Sandbox Per Agent

Claude gets one sandbox, Codex gets another. Clean separation of toolchains and state.

⌨️

Lifecycle from the Dashboard

Create, start, stop, remove, all from keyboard shortcuts. No terminal context-switching.

Workflow

From zero to PR in six keystrokes

The core loop, from creating a worktree to shipping a pull request.

1

Register

biomelab auto-adds your current repo on first launch. Or press a to add more.

2

Create worktrees

Press c, type a branch name, press Enter. Done.

3

Open in terminal

Press Enter. Attaches to the sandbox session or opens a local terminal. Start your agent there.

4

Agent auto-detected

biomelab picks up the running agent. Start working with it! The card will turn green.

5

Send PR

Press P to push and create the PR in one step. Handles dirty state and multi-remote.

6

PR status flows in

PR #42 fix auth (open) ✓. CI checks, merge status, all on the card.

Keyboard-First

Mouse optional, keyboard mandatory

Every action is a single keystroke. Toggle mouse mode with m when you want it.

dDelete worktree
